### **Thao Tác 1: Xem Global KPIs (30 Ngày)**

**Mục tiêu:** Theo dõi hiệu suất toàn nền tảng

### **Bước 1: Truy cập Dashboard**

### **Bước 2: Xem 4 KPI Chính**

Bạn sẽ thấy 4 metrics tính toán trong **30 ngày gần nhất**:

```
┌─────────────────────────────────────────────────────────────┐
│ 1️⃣ TỔNG SỐ INFLUENCERS (Total Influencers)                 │
│    Định nghĩa: Tổng số influencer đã đăng ký tính đến now   │
│    VD: 2,450 influencers                                     │
│    Công thức: COUNT(users) all-time                         │
│                                                              │
│ 2️⃣ INFLUENCER MỚI (New Influencers)                        │
│    Định nghĩa: Số influencer mới đăng ký trong 30 ngày     │
│    VD: 245 influencers mới                                  │
│    Công thức: COUNT(users WHERE created_at in last 30d)    │
│                                                              │
│ 3️⃣ TỶ LỆ HOẠT ĐỘNG (Activity Rate) - %                    │
│    Định nghĩa: % influencer có ≥1 lần submit content       │
│    VD: 68.5%                                                │
│    Công thức: (Hoạt động / Tổng số) × 100                  │
│    → Hoạt động = Có submit content trong 30 ngày           │
│                                                              │
│ 4️⃣ TỶ LỆ RỜI BỎ (Churn Rate) - %                          │
│    Định nghĩa: % influencer không hoạt động trong 30 ngày  │
│    VD: 31.5%                                                │
│    Công thức: 100% - Activity Rate                          │
│    → Nghỉ = Không submit content trong 30 ngày             │
└─────────────────────────────────────────────────────────────┘
```

### **Bước 3: Diễn Giải Kết Quả**

**Ví dụ Scenario:**

```
Ngày: 25/02/2026 (30 ngày trước = 26/01/2026)

KPIs hiển thị:
├─ Tổng số: 2,450 ↑ 2.5% (so với kỳ trước)
├─ Mới: 245 ↑ 5.2%
├─ Activity: 68.5% ↓ 1.3%
└─ Churn: 31.5% ↑ 1.3%

Giải thích:
✅ Platform đang tăng trưởng (2.5% tăng)
⚠️ Nhưng engagement giảm (activity -1.3%)
→ Cần xem xét: Tăng user mới nhưng họ không active?
```

### **Bước 4: So Sánh Với Kỳ Trước**

- **Trend Indicator:** Mũi tên ↑/↓ hiển thị tăng/giảm
- **Threshold:** ±0.5% được coi là bình thường
- **Phép Tính:**
    
    ```
    Kỳ trước = COUNT(users WHERE created_at <= 30 ngày trước)
    Tăng/Giảm % = ((Current - Previous) / Previous) × 100
    ```

### **Bước 3: Hiểu 9 Transfer States**

```
Transfer Status Workflow:

1. PENDING (Chờ xử lý)
   → Batch vừa tạo, đợi xử lý

2. IN_PROGRESS (Đang xử lý)
   → System đang tính toán, validate dữ liệu

3. VALIDATING (Xác thực)
   → Kiểm tra account đích, số tiền

4. PROCESSING (Xử lý thanh toán)
   → Gửi đến ngân hàng

5. COMPLETED (Hoàn tất)
   → ✅ Thanh toán thành công

6. PARTIALLY_COMPLETED (Hoàn tất từng phần)
   → ✅ Một số transfer OK, một số fail

7. FAILED (Thất bại)
   → ❌ Thanh toán thất bại (lỗi hệ thống/ngân hàng)

8. REJECTED (Từ chối)
   → ⛔ Được từ chối (check lại request)

9. CANCELLED (Hủy bỏ)
   → 🚫 Admin hủy batch

Dòng chảy típical:
PENDING → IN_PROGRESS → VALIDATING → PROCESSING → COMPLETED
                                            ↓
                                    (hoặc FAILED)
```

## **📊 HIỂU RÕ CÁC METRICS**

### **Key Metric Definitions**

### **Metric Set A: Platform Metrics**

| Metric | Definition | Formula | Use Case |
| --- | --- | --- | --- |
| **Total Views** | Tổng lượt xem của tất cả videos | SUM(views) all videos | Đánh giá reach |
| **Net Views** | Lượt xem (loại bỏ rejected) | SUM(views) approved + pending | Thực tế reach |
| **Engagement** | Tổng interactions | Likes + Comments + Shares | Đánh giá interaction |
| **Engagement Rate** | % tương tác trên views | (Engagement / Views) × 100 | Compare platforms |

### **Metric Set B: Financial Metrics**

| Metric | Definition | Formula | Use Case |
| --- | --- | --- | --- |
| **Total Ad Spend** | Chi phí quảng cáo | SUM(payments made) | Track budget spend |
| **CPV (Cost Per View)** | Chi phí/lượt xem | Spend / Views | Optimize efficiency |
| **CPE (Cost Per Engagement)** | Chi phí/interaction | Spend / Engagement | Compare ROI |
| **Budget Utilization** | % budget đã dùng | (Spent / Allocated) × 100 | Monitor spend |

### **Metric Set C: Influencer Metrics**

| Metric | Definition | Formula | Use Case |
| --- | --- | --- | --- |
| **Total Influencers** | Số influencer tổng cộng | COUNT(users) | Platform size |
| **New Influencers** | Số mới trong period | COUNT(created in 30d) | Growth tracking |
| **Activity Rate** | % có submit content | (Active / Total) × 100 | Engagement health |
| **Churn Rate** | % không hoạt động | 100% - Activity Rate | Retention health |

### **Công Thức Tính Toán**

### **Trend Analysis**

```
So sánh với kỳ trước (Previous Period):

Kỳ này (Current): 2026-01-26 → 2026-02-25
Kỳ trước (Prior): 2025-12-27 → 2026-01-25

Công thức trend:
Percent Change = ((Current - Previous) / Previous) × 100

Ví dụ:
├─ Current: 1,000 views
├─ Previous: 950 views
└─ Change: (1,000 - 950) / 950 × 100 = 5.26% ↑

Threshold:
✅ ±0.5% = No significant change (normal variation)
🟡 ±0.5% to 5% = Slight trend
🔴 > 5% = Significant trend (investigate)
```

### **Aggregation Logic**

```
Data flow cho metrics:

Raw Events (hourly)
        ↓
MongoDB Collections:
├─ event_analytic_daily
├─ content_analytic_daily
├─ user_event_analytic_daily
└─ transfers
        ↓
Pre-computed Statistics
        ↓
Dashboard Display

Timing:
├─ Real-time: Views, engagement, status
├─ Daily refresh: Approval stats, CPV
└─ Hourly summary: Batch processing status
```

### **Net vs Gross Values**

```
GROSS values: Include rejected content
NET values: Exclude rejected content

Examples:
├─ Gross Video Count: 450 (approved + pending + rejected)
└─ Net Video Count: 410 (approved + pending only)

├─ Gross Views: 1,300,000 (all videos)
└─ Net Views: 1,250,000 (excluding rejected video views)

Why distinction matters:
→ Gross = Potential effort
→ Net = Actual value delivered

Use case:
├─ Gross for: Capacity planning
└─ Net for: Performance analysis
```

## **🎓 COMMON SCENARIOS**

### **Scenario 1: Campaign ROI Analysis**

**Tình huống:** Cần so sánh 3 campaigns về hiệu quả

```
Campaign A: Video = 100, Views = 500K, Spend = 20M, CPV = 40
Campaign B: Video = 150, Views = 450K, Spend = 18M, CPV = 40
Campaign C: Video = 80,  Views = 600K, Spend = 24M, CPV = 40

Analysis:
1. Video Count: Campaign B most productive (150 videos)
2. Views: Campaign C most efficient (600K views)
3. CPV: All equal (40 VND/view) → Similar efficiency
4. Conclusion:
   → Campaign C: Best ROI (highest views, reasonable spend)
   → Campaign A: Good balanced performance
   → Campaign B: Most videos, but lower quality (450K views)
```

### **Scenario 2: Platform Performance Comparison**

**Tình huống:** Xác định platform nào nên focus

```
Data:
YouTube:  200 videos, 1.2M views, 85% approval → CPV = 35
Facebook: 150 videos, 600K views, 80% approval → CPV = 45
TikTok:   300 videos, 1.5M views, 75% approval → CPV = 25

Recommendation:
1. Best ROI: TikTok (CPV 25 = lowest cost)
2. Best Quality: YouTube (85% approval)
3. Scale: TikTok (300 videos, 1.5M views)

Action:
→ Increase TikTok investment (lowest CPV)
→ Improve YouTube quality (high approval)
→ Monitor Facebook (mid metrics, lowest views)
```

### **Scenario 3: Quality Improvement**

**Tình huống:** Approval rate drop từ 85% → 75%

```
Investigation:
1. Check rejection reasons:
   → Low Quality: 50% (was 30%)
   → Other: 50% (was 70%)

2. Root cause:
   → New creators joined?
   → Training needed?
   → Quality standards changed?

3. Action plan:
   ├─ Review low quality videos
   ├─ Create training material
   ├─ Set clear guidelines
   ├─ Add review process
   └─ Monitor weekly for improvement

4. Target: Return to 85% within 4 weeks
```
